
# 💡 このプロジェクトの概要
Spring Securityを使用した、一般的なログイン・ユーザー登録機能のサンプルです。

## 📁 フォルダ構成と役割
- `src/main/java/...`
  - `DemoApplication.java`: アプリの起動クラス
  - `SecurityConfig.java`: 🔒 セキュリティ設定（ログインの許可・制限）
  - `HelloController.java`: 🎮 画面遷移やTOP画面を制御するコントローラー
  - `UserController.java`: 🎮 ユーザー登録やログイン処理のコントローラー
  - `User.java`: 📦 ユーザー情報を表すデータモデル（Entity）
  - `UserRepository.java`: 💾 データベースへアクセスするインターフェース
  - `UserDetailsServiceImpl.java`: 👤 DBからユーザー情報を探して認証するサービス
- `src/main/resources/...`
  - `style.css`: 🎨 ログイン画面などの見た目を整えるスタイルシート
  - `templates/login.html`: 📄 ログイン画面の見た目
  - `templates/register.html`: 📄 新規登録画面の見た目
  - `mypage.html`: 🏠 ログイン後に表示されるマイページ
  - `admin.html`: 👑 管理者専用のページ
  - `application.properties`: ⚙️ データベース接続やポート番号などの全体設定

# 🔑 ログイン機能のファイル構成と役割

このプロジェクトにおける各ファイルの役割を「画面」「制御」「データ」の3つに分類してまとめています。

---

## 1. 🎨 画面（見た目）を担当する部分

ユーザーがブラウザで直接目にする、デザインや入力フォームを構成するファイル群です。

* **`login.html` / `register.html`**
    * ユーザーが実際に目にする「ログインフォーム」や「新規ユーザー登録フォーム」です。
* **`style.css`**
    * ログインボタンの色、入力欄の幅、全体の配置など、画面のデザインを綺麗に整えます。

---

## 2. 🎮 制御（仕組み・セキュリティ）を担当する部分

画面の切り替えや、システム全体のアクセス制限など、アプリケーションの「動き」をコントロールする中心部です。

* **`SecurityConfig.java`**
    * > 💡 **重要ファイル**
    * 「ログイン画面は誰でも見られるようにする」「マイページはログインした人（認証済み）だけ許可する」といった、**セキュリティのルール**を厳密に決めるファイルです。
* **`UserController.java` / `HelloController.java`**
    * ユーザーがボタンを押したとき、その入力を受け取って次の画面（`mypage.html` など）へ案内する**受付係（コントローラー）**です。

---

## 3. 💾 データ（ユーザー情報）を担当する部分

データベース（DB）とのやり取りや、ユーザー情報の管理・チェックを行う裏方の処理です。

* **`User.java`**
    * ID、パスワード、名前といった「ユーザー情報の項目」を定義するファイル（Entity）です。
* **`UserRepository.java`**
    * データベースに対して、ユーザーデータの保存や検索を行うための**専用の窓口**です。
* **`UserDetailsServiceImpl.java`**
    * ⚙️ **認証の核心部分**
    * ユーザーが入力したログインIDを元に、`UserRepository` を使って「本当にこのユーザーはDBに存在するか？」をチェックする重要なサービスです。
